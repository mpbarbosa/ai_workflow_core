### 2. Promise Overhead

- **File**: `.workflow_core/bin/ai-workflow.js`  
  **Severity**: LOW  
  **Dimension**: Promise Overhead  
  **Issue**: Uses an async IIFE with sequential awaits (`await access(...); await import(...)`) where these could run concurrently if independent.  
  **Fix**:  
  _Before:_  
  ```js
  await access(parentCliPath);
  await import(pathToFileURL(parentCliPath).href);
  ```
  _After:_  
  ```js
  await Promise.all([
    access(parentCliPath),
    import(pathToFileURL(parentCliPath).href)
  ]);
  ```
  **Impact**: Minor; only relevant if both are independent and performance is critical.

- **File**: `.workflow_core/src/loader.ts`  
  **Severity**: LOW  
  **Dimension**: Promise Overhead  
  **Issue**: Several exported `async` functions (e.g., `loadPromptRoles`, `loadPersonas`) are thin wrappers around a single `await` call.  
  **Fix**:  
  _Before:_  
  ```ts
  export async function loadPromptRoles(filePath: string): Promise<PromptRolesConfig> {
    const raw = await readYamlFile(filePath);
    // ...
  }
  ```
  _After:_  
  ```ts
  export function loadPromptRoles(filePath: string): Promise<PromptRolesConfig> {
    return readYamlFile(filePath).then(raw => {
      // ...
    });
  }
  ```
  **Impact**: Very minor; async/await overhead is negligible unless on a hot path.

### 7. Error Handling

- **File**: `.workflow_core/bin/ai-workflow.js`  
  **Severity**: HIGH  
  **Dimension**: Error Handling  
  **Issue**: Top-level async IIFE does not have a global unhandledRejection handler; errors are only caught within the IIFE.  
  **Fix**:  
  _Before:_  
  ```js
  (async () => {
    try { ... } catch (error) { ... }
  })();
  ```
  _After:_  
  ```js
  process.on('unhandledRejection', (err) => {
    console.error('Unhandled rejection:', err);
    process.exit(1);
  });
  (async () => { ... })();
  ```
  **Impact**: Prevents silent process crashes on unhandled promise rejections.

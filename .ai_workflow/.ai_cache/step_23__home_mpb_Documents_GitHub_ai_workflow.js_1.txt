### 2. Repeated YAML Parsing Without Caching

**File:** `.workflow_core/src/loader.ts`  
**Line(s):** `readYamlFile`, `loadPromptRoles`, `loadPersonas`  
**Severity:** Medium  
**Impact:** If config files are loaded multiple times (e.g., in a long-running process or CLI invoked repeatedly), parsing the same YAML files repeatedly is wasteful.

**Current:**  
```ts
export async function loadPromptRoles(filePath: string): Promise<PromptRolesConfig> {
  const raw = await readYamlFile(filePath);
  // ...
}
```
**Recommendation:**  
Implement in-memory caching for parsed YAML files to avoid redundant parsing.

**Before:**
```ts
export async function loadPromptRoles(filePath: string): Promise<PromptRolesConfig> {
  const raw = await readYamlFile(filePath);
  // ...
}
```
**After:**
```ts
const yamlCache = new Map<string, unknown>();
async function readYamlFile(filePath: string): Promise<unknown> {
  if (yamlCache.has(filePath)) return yamlCache.get(filePath);
  const raw = await fsPromises.readFile(filePath, 'utf8');
  const parsed = parseYaml(raw);
  yamlCache.set(filePath, parsed);
  return parsed;
}
```
**Benefit:**  
Reduces I/O and parsing overhead for repeated config loads.
